# ORVIXA PRODUCT & UX CONSTITUTION
*Universal AI Learning & Interview Copilot*

---

## 1. CORE IDENTITY

> "An AI teacher that appears exactly where learning happens."

Orvixa is built on a single, uncompromising principle:
**The user should never leave their learning environment to receive intelligent help.**
Orvixa goes to the learner; the learner never goes to Orvixa.

---

## 2. THE 6 PLATFORM LAWS

1. **Invisible by Default:** No floating badges, blinking widgets, notifications, or promotional UI. Orvixa stays hidden until explicitly summoned.
2. **Summon Over Auto-Open:** Default activation is triggered exclusively via configurable keyboard shortcut (`Ctrl+K` / `Cmd+K`) or Right Click -> `Ask Orvixa`.
3. **On-Demand Context Capture:** Never read the screen automatically. Screen understanding happens *only* after explicit user action.
4. **Adaptive Response Layout:** Responses are composed of small, focused "Adaptive Learning Cards" matching the domain context. No long wall-of-text paragraphs.
5. **Intent-First Teaching:** Automatically detect if the learner wants a direct answer, a hint, deep teaching, or an interview run.
6. **Platform Heroism:** The host website (LMS, GitHub, Docs, LeetCode) remains the hero. Orvixa acts strictly as a quiet intelligence layer beside it.

---

## 3. RESPONSE MATRIX & INTENTS

| Intent Mode | Description | Adaptive Cards Used |
| :--- | :--- | :--- |
| **Answer Only** | Direct, concise solution | `Formula`, `Meaning`, `Ideal Answer` |
| **Hint Only** | Socratic progressive disclosure | `Socratic Hint Ladder`, `Common Mistakes` |
| **Explain** | 1-sentence breakdown of logic | `Concept`, `Visualization`, `Usage` |
| **Teach Deeply** | Comprehensive conceptual walkthrough | `Concept`, `Numerical`, `Practice` |
| **Interview Mode** | Mock roleplay dialogue | `Question`, `Weak Answer`, `Feedback` |
| **Revision Mode** | Flashcards or key highlights | `Key Metrics Grid`, `Timeline` |

---

## 4. UX & DESIGN LAWS

- **Visible Pixel Purity:** Welcome cards, promotional texts, and upgrade banners are strictly forbidden. Every element on the side panel must support the immediate learning objective.
- **Side Panel Mechanics:** The panel sits on the right side of the screen at a default width of 35% (resizable 25%–50%). Toggles open/closed under 150ms.
- **Escape Hatch:** Pressing `Esc` immediately collapses the side panel and aborts any active streaming AI requests.
- **Privacy Control:** Compact screen-scan icon in BottomBar. Captures screen context ONLY after explicit user click.
